Riprendiamo la nostra Todo List

Proviamo ad immaginare ad assegnare delle proprietà ai nostri task, ad esempio uno stato (fatto/da fare).

Nelle App avremo sempre bisogno di 2 posti da aggiornare (UI/dati) e un posto per recuperare i dati

La UI è temporanea, ci giochiamo, la cambiamo, cancelliamo, modifichiamo ...

Abbiamo bisogno di strutture dati per memorizzare i nostri dati e poi visualizzarli quando richiesto e nel formato richiesto.

Cominciamo dalla UI

Abbiamo bisogno di una select per selezionare tre opzioni, per permetterci di visualizzare i task completati, quelli da fare, oppure tutti, quindi cominciamo da li e associamogli un evento change per monitorare i cambiamenti del valore scelto dall'utente.

Aggiornare la UI?

Quello che vogliamo che accada al cambiamento della selezione è visualizzare i task in base al valore selezionato, quindi dobbiamo prima di tutto fare in modo di poter cambiare questo stato, partendo sempre dalla UI aggiungiamo un tasto ad ogni task che ci permette di cambiare il suo stato.

Cambiare i dati

Alla pressione del tasto dobbiamo cambiare lo stato del task inerente a quel tasto (collegamento tra UI e dati). Come facciamo a sapere a quale task si riferisce quel tasto? Usiamo gli attributi e gli id nelle strutture dati. Gli id sono importanti, servono per identificare univocamente gli oggetti, gli elementi, devono essere UNIVOCI, non possiamo basarci sulle altre proprietà (titolo, descrizione, etc...)..

Un po' di funzioni comode

selezionando un elemento dal DOM con .parent() trovo il padre!

<div id='container'> <span id='content'>Content</span> </div> ..... $('#content').parent().attr('id'); // "container"

E' possibile esplodere le stringhe in un array specificando un separatore

var a = "task_2"; a.split('_'); // ['task','2']; a.split('_')[1]; // '2';

If ternario

Esiste una versione compatta dell'if molto utile da utilizzare per costruire stringhe o per assegnare valori a variabili basati solamente su una condizione:

var status = true; // if normale if(status==true) //oppure if(status) che è equivalente toggle_status=false; else toggle_status=true; // if ternario var toggle_stato = (stato?false:true); // altro esempio var i = 2; console.log("Il numero i è " + (i%2?"dispari":"pari"));
