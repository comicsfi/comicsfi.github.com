Come lo usiamo

Includiamo i file jq.mobi.js (per farla breve è un sostituto tarato per il mobile di jquery), e jq.ui.js che aggiunge un layer per facilitarci la vita fornendoci funzionalità di alto livello (pannelli, history, sideBar, manuBar, navBar, showMask, plugins).

Includiamo i css di default e quello delle icone se le necessitiamo


<script type="text/javascript" charset="utf-8" src="jq.mobi.js"></script> <script type="text/javascript" charset="utf-8" src="jq.ui.js"></script> <link rel="stylesheet" type="text/css" href="css/icons.css"> <link rel="stylesheet" type="text/css" href="css/jq.ui.css">

Viewport

Pixel is not pixel!!! Nei dispositivi mobili, un pixel CSS NON corrisponde al pixel del dispositivo !!! Nei display retina ad esempio per ogni pixel abbiamo 4 subpixel (ognuno puo' visualizzare un proprio colore ovviamente)!!
Se visualizziamo un sito con un container principale di 900px su iphone viene visualizzato a tutto schermo nonostante la larghezza orizzontale sia 320px, questo perchè subisce uno zoom-out, ovvero si basa su un'altra larghezza ovvero la viewport (che per iphone ha un default di 980px, quindi scala fino a raggiungere la dimensione corretta). Usando il tag viewport impostiamo noi la grandezza del viewport!!

<!-- in mobile pixel is not a pixel --> <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
